# 11 個彩色 tag 的推斷規則表

> 規格出處：plan v3 §4（11 curated tags）+ §11（parser architecture）。
> 用途：parser 讀醫院官網內容後，依此表自動套用 multi-select tags 到 Notion。
> 規則改動需同步更新 v3 plan + 重跑 parser pilot 對照舊 12 家 baseline。
>
> 最近更新：2026-05-23（v3.2 架構轉向初稿）

---

## 0. 證據強度分級（每個 tag 用其中一級）

| 強度 | 意義 | Parser 行為 |
|---|---|---|
| **A. 制度性事實** | 衛福部/醫策會等公開認證（醫學中心、教學醫院） | Cross-check `data/hospitals-reference.md` 才能自動套用；網頁出現「醫學中心」字樣但不在清單上 → 不套，flag 為 review |
| **B. 文本明證** | 招募頁/福利頁明確寫出（夜班費 \d+、提供宿舍、簽約金 \d+） | Regex/關鍵字命中即套，紀錄 evidence 字串到 audit log |
| **C. 主觀判斷** | 「工作單純」「重視教學」這類需要 Evelyn 讀完整段才能判定的 | Parser **不自動套**，僅 flag「可能符合」並列出觸發語句，等 Evelyn 在 diff report 勾選 |

每個 tag 下面標明其等級。Level A/B 才可全自動；Level C 必須人工確認。

---

## 1. 生活/班別（blue）

### 1.1 工作單純 — Level C（主觀）

- **觸發 hints**（命中後 flag for review，不自動套）：
  - 「工作單純」「業務單純」直接出現
  - 「無化療」「無 TPN」「無無菌調配」「無細胞毒性藥物」
  - 職務範圍只列「調劑」「發藥」「住院/門診配藥」而**不**提及 TPN/化療/IVA/臨床照會
- **排除條件**：
  - 出現「臨床藥師」「TPN 配製」「化療配製」「處方審核」「藥物諮詢門診」之一 → 不套
- **Anchor 已建檔範例**：三總北投分院（「無化療無 TPN，工作單純」）、新國民醫院、秉坤婦幼

### 1.2 免/少輪班 — Level B（文本明證）

- **觸發條件**（任一命中即自動套）：
  - 「免大夜」「免小夜」「無小夜」「免輪班」直接出現
  - Regex：`大夜.{0,3}每月\s*0\s*[-至到~]\s*2\s*[天班]` 或 `小夜.{0,5}免`
  - 「以日班為主」+ 「不需輪大夜」
- **排除條件**：
  - 同時出現「需輪三班」「日/晚/大夜輪值」 → 不套
- **與 1.3 的差別**（很重要，不能誤套）：
  - 免/少輪班 = 整體輪班負擔減輕，**可能仍有少量大夜**（北榮桃園每月 0-2 天大夜也算）
  - 無大夜 = **零大夜**，週末日班 / 假日班可有
- **Anchor**：北榮桃園分院（「免小夜·大夜每月 0-2 天」）、永和耕莘

### 1.3 無大夜 — Level B（文本明證）

- **觸發條件**：
  - 「無大夜」「不需大夜」「免大夜」直接出現（**注意**：「免大夜」也可同時觸發 1.2，兩個 tag 可並存）
  - 班別表只列「日班」「日班 + 晚班」「日班 + 假日」**而完全不出現**「大夜」「23:00」「凌晨」字樣
  - Regex：上下班時間若全部為 `0[7-9]:.{0,3}1[0-9]:` 或 `08:.{0,3}17:` 類日班區間
- **排除條件**：
  - 任何提及「大夜」「凌晨」「23:00-」「24 小時輪值」 → 不套
- **Anchor**：新國民（日班 + 週六半天）、秉坤婦幼（日/晚/假日，無大夜）

### 1.4 夜班津貼優渥 — Level B（文本明證）

- **觸發條件**：
  - Regex：`夜班費\s*[\d,]+\s*元?` 抓出數字，**≥ 1000 元/班** → 套
  - Regex：`夜班\s*津貼\s*[\d,]+` 同上
  - 描述如「夜班費業界 top」「夜班補助優於同業」直接觸發 + flag for review
- **排除條件**：
  - 數字 < 1000 → 不套
  - 完全無夜班（已套 1.3 無大夜 + 無小夜）→ 不套（沒有夜班何來津貼）
- **Anchor**：（尚無，待官網資料）

## 2. 規模/成長（purple）

### 2.1 醫學中心級 — Level A（制度性事實）

- **觸發條件**：
  - 醫院名稱必須出現在 `data/hospitals-reference.md` §一. 醫學中心（23 家）名單
  - 網頁是否寫「醫學中心」字樣**不是主要依據**（自稱不算）
- **排除條件**：
  - 不在衛福部清單 → 不套（即使網頁寫「醫學中心級服務」也不套）
- **Anchor**：國泰、萬芳（兩家已建檔；其餘 21 家待建）

### 2.2 教學醫院 — Level A（制度性事實）

- **觸發條件**：
  - 衛福部「教學醫院評鑑」合格清單中
  - 已是醫學中心 → 自動含教學醫院（醫學中心評鑑包含教學）
- **排除條件**：
  - 不在清單 → 不套
- **Anchor**：安南醫院（區域教學）；所有已建檔醫學中心

### 2.3 重視教學 — Level C（主觀，需文本佐證）

- **觸發 hints**：
  - 「PGY」「PGY-2」字樣
  - 「臨床訓練計畫」「住院醫師制」「藥師訓練制度」
  - 「教學品質佳」「導師制」「mentor」
  - 「醫學中心模式」（用於非醫學中心）
- **排除條件**：
  - 僅是醫學中心 / 教學醫院身分（已套 2.1/2.2）但無上述細節 → 不套（重複資訊無價值）
- **Anchor**：輔大附醫（「醫學中心模式·PGY」）

### 2.4 全面藥事訓練 — Level C（主觀）

- **觸發 hints**：
  - 「全面藥事訓練」直接出現
  - 列出「住院 / 門診 / 急診 / TPN / 化療 / 臨床 / 諮詢」**3 種以上**輪科
  - 「輪科訓練」+ 列舉
- **排除條件**：
  - 工作內容只寫「調劑」單一項 → 不套
- **Anchor**：（待 parser 跑出來看）

### 2.5 外派進修機會 — Level B（文本明證）

- **觸發條件**：
  - 「外派進修」「公費進修」「補助進修」「海外研修」
  - 「派外受訓」「他院研修」
- **排除條件**：
  - 只寫「鼓勵繼續教育」這類空話 → 不套
- **Anchor**：（待 parser 跑）

## 3. 加分福利（green）

### 3.1 簽約金 — Level B（文本明證）

- **觸發條件**：
  - 「簽約金」「久任獎金」「到職獎金」字樣出現
  - 通常有附金額（如「簽約金 20 萬」）
- **排除條件**：
  - 僅寫「另有獎金」未指明簽約類 → 不套
- **Anchor**：北榮桃園、永和耕莘、安南醫院

### 3.2 提供宿舍 — Level B（文本明證）

- **觸發條件**：
  - 「提供宿舍」「宿舍可申請」「外地免費宿舍」「員工宿舍」
  - 福利 chip / 列表中有「宿舍」項
- **排除條件**：
  - 「宿舍限輪值三班、非設籍雙北者申請」這類**限制條件**：仍套（有條件提供算提供），但 audit log 註明限制
- **Anchor**：輔大附醫、三總北投、北榮桃園、永和耕莘等

## 4. Parser 實作備註

1. **匹配優先順序**：先跑 Level A（cross-check 清單）→ 再跑 Level B（regex/關鍵字）→ Level C 一律 flag for review，不自動套。
2. **多 tag 並存允許**：一家醫院可同時是 醫學中心級 + 教學醫院 + 重視教學 + 全面藥事訓練（同 group 多 tag 沒問題）。
3. **Audit log 必須記錄**：每個套用的 tag 旁邊記下「觸發字串/regex 命中位置」，存到 `data/parser-audit/{hospital_id}-{date}.json`，diff report 引用。
4. **舊 12 家不重跑**：parser 從第 13 家起套用此規則表；現有 12 家保留 Evelyn 人工判斷結果。
5. **規則升級流程**：發現誤套 / 漏套 → 修這份 .md → 同步更新 `parsers/tag-rules.json` → 跑 regression test 對 12 家 baseline。
